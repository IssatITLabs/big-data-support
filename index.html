Objectifs du cours

Ce cours vise à initier les étudiants au langage **SQL (Structured Query Language)**, outil fondamental pour la gestion et l’interrogation des bases de données relationnelles.

À la fin de ce module, les étudiants seront capables de :

  • Comprendre la **structure** et les **composantes** d’une base de données SQL.
  • Maîtriser les commandes du DDL pour **définir la structure** des tables.
  • Maîtriser le DML et le DQL pour **manipuler et interroger** efficacement les données.
  • Utiliser les fonctions d’agrégation et les jointures pour **synthétiser et relier** les informations.

Section 1 — Introduction & Structure de SQL

Qu’est-ce que SQL ?

Le **SQL (Structured Query Language)** est un langage standard et déclaratif conçu pour gérer et manipuler les données dans les systèmes de gestion de bases de données relationnelles (SGBDR).

Importance de SQL

SQL est la pierre angulaire de la gestion de données dans l'industrie. Il est utilisé par toutes les grandes plateformes SGBDR (MySQL, PostgreSQL, Oracle, SQL Server, etc.) et est indispensable pour les ingénieurs, les analystes et les développeurs.

Structure d’une BD SQL (Exemple)

Nous utiliserons l’exemple de la base de données db_etudiants dont le schéma relationnel est le suivant :

**Clé primaire (PK)** : Colonne ou ensemble de colonnes garantissant l'**unicité** de chaque ligne.
**Clé étrangère (FK)** : Colonne faisant référence à une clé primaire d'une autre table, établissant ainsi la **relation** entre les tables.

Section 4 — Fonctions d’Agrégation et Groupement

Les fonctions d’agrégation (SUM, AVG, COUNT, MAX, MIN) effectuent un **calcul sur un ensemble de lignes** pour retourner une valeur unique.

1. Fonctions d'Agrégation Simples

Exemple 1: Calculer le total du volume horaire de tous les cours (SUM).

Exemple 2: Compter le nombre d'étudiants inscrits dans au moins un cours (COUNT).

2. Groupement des Résultats (GROUP BY)

La clause **GROUP BY** est utilisée pour regrouper les lignes ayant les mêmes valeurs dans une colonne, permettant aux fonctions d'agrégation d'opérer sur chaque groupe.

Exemple 3: Calculer la note moyenne par cours.

3. Filtrage sur les Groupes (HAVING)

La clause **HAVING** filtre les résultats du regroupement après l'agrégation, contrairement à WHERE qui filtre les lignes avant l'agrégation.

Exemple 4: Afficher uniquement les cours dont la note moyenne est supérieure à 15.

Section 5 — Jointures (Relier les Tables)

Les **Jointures** (JOIN) permettent de combiner les colonnes de deux ou plusieurs tables en se basant sur une colonne de relation (généralement la Clé Étrangère).

1. INNER JOIN (Jointure Interne)

Affiche uniquement les lignes ayant des correspondances dans **les deux tables**.

Exemple 1 : Afficher les noms des étudiants avec les cours auxquels ils sont effectivement inscrits.

2. LEFT (OUTER) JOIN (Jointure Gauche)

Affiche **toutes les lignes de la table de gauche** (première mentionnée) et les correspondances de la table de droite. Les lignes sans correspondance de droite affichent NULL.

Exemple 2 : Afficher tous les étudiants, même ceux qui n'ont pris aucun cours.

**Autres jointures :** **RIGHT JOIN** (similaire au LEFT JOIN, mais pour la table de droite) et **FULL JOIN** (toutes les lignes des deux tables).
